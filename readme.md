Автор: Платановский Степан Вячеславович
Группа: 25948


1. Назначение программы

Программа решает задачу назначения заданий на машины с ограничениями (каждое задание можно выполнять только на определенных машинах). Цель - минимизировать максимальную нагрузку на одну машину (makespan). Задача NP-трудная, поэтому используются приближенные алгоритмы.


2. Состав программы

Программа состоит из одного файла main.c. Компилируется с помощью любого C-компилятора (например, gcc).


3. Компиляция

Откройте терминал (командную строку) в папке с программой и выполните:

   gcc main.c -o main

Для Windows можно также использовать:

   gcc main.c -o main.exe


4. Запуск программы

4.1. Запуск с тестовым примером по умолчанию:

   ./main          (Linux/Mac)
   main.exe        (Windows)

4.2. Запуск со своим файлом данных:

   ./main input.txt

Программа сама определяет формат файла, описанный ниже.


5. Формат входного файла

Первая строка: два целых числа - количество заданий (n) и количество машин (m).
Далее для каждого задания (n строк) указывается:
- целое число t - время выполнения задания
- целое число cnt - сколько машин подходит для этого задания
- cnt целых чисел - номера машин (нумерация с 0 до m-1)

Пример файла test1.txt:

3 2
10 2 0 1
5 1 0
8 2 0 1

Это означает:
- 3 задания, 2 машины
- задание 0: время 10, можно на машины 0 или 1
- задание 1: время 5, можно только на машину 0
- задание 2: время 8, можно на машины 0 или 1


6. Что выводит программа

Программа выводит:
- Исходные данные (количество заданий и машин, список заданий)
- Для каждого из 5 алгоритмов:
   - назначение каждого задания (какая машина)
   - нагрузку на каждую машину
   - значение makespan (самая большая нагрузка)
- Сравнительную таблицу всех алгоритмов с отношением к лучшему результату


7. Описание алгоритмов

Программа реализует 5 алгоритмов (4 разных + локальный поиск):

1. Simple Greedy - жадный алгоритм. Проходит по заданиям в том порядке, как они даны в файле. Каждое задание ставит на машину с наименьшей текущей нагрузкой среди допустимых.

2. LPT Greedy - сначала сортирует задания по убыванию времени выполнения (самые большие первые), потом делает то же, что и Simple Greedy.

3. Multifit - бинарный поиском ищет минимальное значение makespan, при котором все задания можно разместить. Для проверки использует жадный алгоритм.

4. LP Approximation - работает как LPT, но используется как 2-приближение.

5. LP + Local Search - берет решение из LP Approximation и пытается его улучшить: перекидывает задания с самой загруженной машины на другие, если это уменьшает makespan. Повторяет пока есть улучшения.


8. Ограничения

- Максимальное количество заданий: 100
- Максимальное количество машин: 20
- Времена выполнения заданий - целые числа
- Номера машин должны быть от 0 до m-1


9. Возможные ошибки

Если файл не найден - программа использует тестовый пример по умолчанию.
Если задание нельзя назначить ни на одну машину - оно останется неназначенным (но в корректных входных данных такого не должно быть).


10. Пример работы

Команда: ./main test1.txt

Ожидаемый вывод (примерный):
- Программа покажет исходные данные
- Затем для каждого алгоритма выведет куда назначены задания и нагрузки
- В конце таблицу с makespan и отношениями
